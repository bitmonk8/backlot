### Stage 5: Epic (`gate/src/stage/epic.rs`)

Replace the stub. Three tests exercising epic's recursive orchestrator.

**Project generation (not fixtures):**

Epic tests need a Rust project with a known bug, a git repo, and an `epic.toml`. These are generated programmatically in the scratch dir — not committed as static fixtures. This ensures the project is always fresh and matches the current toolchain.

```rust
/// Generate a minimal Rust project with a deliberate bug in the scratch dir.
/// Returns the project root path.
fn generate_test_project(scratch_dir: &Path) -> std::io::Result<PathBuf>;
```

The generated project:

**`Cargo.toml`:**
```toml
[package]
name = "gate-test-project"
version = "0.1.0"
edition = "2024"
```

**`src/lib.rs`:**
```rust
/// Add two numbers.
pub fn add(a: i32, b: i32) -> i32 {
    a - b  // BUG: subtraction instead of addition
}
```

**`tests/basic.rs`:**
```rust
use gate_test_project::add;

#[test]
fn test_add() {
    assert_eq!(add(2, 3), 5);
}
```

After writing the files:
1. Run `cargo check` in the project dir to verify the project compiles (the bug is logic, not syntax).
2. Run `git init`, `git add -A`, `git commit -m "initial"` to create a clean git repo. Configure git user.name and user.email for the repo if needed.

**`epic.toml`:**
```toml
[project]
name = "gate-test-project"

[models]
fast = "fast"
balanced = "balanced"
strong = "strong"
```

Check `epic --help` and the epic design doc for the exact `epic.toml` format. The above is illustrative — adjust to match the real format.

**Tests:**

1. **`leaf-task`** — Run epic on the generated project. The task: fix the bug so tests pass. Assert:
   - Exit 0 (epic completed successfully)
   - Run `cargo test` in the project dir → exit 0 (the bug is fixed)
   - `.epic/state.json` exists (state was persisted)
   - This is the primary oracle test — epic's value is measured by whether `cargo test` passes after it runs.

2. **`status`** — After the leaf-task run, invoke `epic status` in the same project dir. Assert: exit 0, output contains status information about the completed run.

3. **`resume-completed`** — After the leaf-task run, invoke `epic resume` in the same project dir. Assert: exit 0, no re-execution (the run is already complete). Verify by checking that the output indicates completion without new work.

**Test ordering:** These three tests must run sequentially — status and resume depend on the state from leaf-task.

**Timeout:** Epic's default timeout is 600s (from D1). The leaf-task test is the most expensive test in the entire suite.
