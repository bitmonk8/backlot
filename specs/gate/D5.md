## What to Build

### `gate/src/runner.rs`

Binary discovery and stage orchestration.

#### Binary discovery

```rust
pub struct BinaryPaths {
    pub flick: PathBuf,
    pub lot: PathBuf,
    pub reel: PathBuf,
    pub vault: PathBuf,
    pub epic: PathBuf,
    pub mech: PathBuf,
}

impl BinaryPaths {
    /// Look up the path for a given stage's binary.
    pub fn for_stage(&self, stage: Stage) -> &Path;
}

/// Locate all required binaries.
/// Resolution order:
/// 1. Explicit bin_dir if provided
/// 2. Workspace target/debug/ (derived from gate's own exe path or CARGO_MANIFEST_DIR)
///
/// On Windows, binary names have .exe suffix.
/// Returns Err listing all missing binaries (not just the first).
pub fn discover_binaries(bin_dir: Option<&Path>) -> Result<BinaryPaths, DiscoveryError>;

pub struct DiscoveryError {
    pub missing: Vec<String>,  // names of missing binaries
    pub search_dir: PathBuf,   // where we looked
}
```

Binary names: `flick`, `lot`, `reel`, `vault`, `epic`, `mech` (plus `.exe` on Windows).

For workspace detection: `CARGO_MANIFEST_DIR` is set during `cargo run` / `cargo test`. At runtime, derive from `std::env::current_exe()` — gate lives in `target/debug/gate(.exe)`, so `target/debug/` is the parent dir. The `--bin-dir` flag overrides both.

#### Stage runner

```rust
/// A stage implementation. Each stage module (flick.rs, lot.rs, etc.) provides
/// a function matching this signature.
pub type StageFn = fn(ctx: &StageContext) -> Vec<TestResult>;

pub struct StageContext {
    pub binaries: BinaryPaths,
    pub config: GateConfig,
    pub scratch_dir: PathBuf,    // per-stage subdirectory within the run dir
    pub output_dir: PathBuf,
}

/// Run all stages according to config filters.
/// Creates scratch dir, runs filtered stages in order, collects results,
/// prints summary, writes JSON if verbose, cleans up scratch on success.
/// Returns exit code (0 = all pass, 1 = any fail, 2 = prereq fail).
pub fn run(config: GateConfig) -> i32;
```

`run()` orchestrates:
1. Discover binaries → exit 2 on failure
2. Create scratch run dir
3. For each stage where `config.should_run(stage)`:
   a. Call the stage function with a StageContext
   b. Collect TestResults into StageResult
   c. If any test is `Fail`, continue to next stage (don't abort — run all requested stages)
4. Print summary table (always)
5. Write `results.json` if verbose
6. Save transcripts if verbose
7. Clean up scratch dir (unless keep_scratch or any failure)
8. Return exit code

#### Stage registry

A simple mapping from `Stage` to `StageFn`. For now, all stages are stubs that return an empty `Vec<TestResult>` — the actual implementations come in D6–D8. The important thing is the runner framework is complete and tested.

```rust
fn stage_fn(stage: Stage) -> StageFn {
    match stage {
        Stage::Flick => stage::flick::run,
        Stage::Lot => stage::lot::run,
        // ...
    }
}
```

## TDD: Tests to Write First

### Binary discovery tests

For tests that check file existence, create a temporary directory (under `target/gate-scratch/`, not system temp) and populate it with dummy files.

1. `discover_explicit_dir_all_present` — create temp dir with all 6 binary files (empty files), call `discover_binaries(Some(dir))` → Ok with correct paths

3. `discover_explicit_dir_empty` — empty dir → Err listing all 6 missing
4. `discover_reports_search_dir` — Err includes the directory that was searched
5. `binary_names_platform_suffix` — on Windows, expected names include `.exe`; on Unix, they don't

### Stage filtering tests

6. `run_all_stages_no_filter` — config with no `only`/`from` → all 6 stages called (verify via stage result count)
7. `run_only_one_stage` — `only: Some(Stage::Reel)` → exactly 1 stage result

9. `stages_run_in_order` — verify stage results appear in dependency order
10. `stage_context_has_correct_scratch_dir` — each stage gets its own subdirectory within the run dir

### Exit code tests

11. `exit_0_when_all_pass` — all stages return Pass results → exit code 0
12. `exit_1_when_any_fail` — one stage has a Fail → exit code 1
