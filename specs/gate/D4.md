# D4: Subprocess Execution and Scratch Directories

## Context

Gate is a binary-only E2E test harness for the backlot workspace. Full design: `specs/GATE.md`. D1 established types (including `CommandResult`).

## Dependencies

D1 (types — `CommandResult`).

## What to Build

### `gate/src/exec.rs`

Subprocess execution with timeout enforcement.

```rust
/// Run a command, capturing stdout, stderr, exit code, and wall-clock duration.
/// If timeout expires, the child is killed and a timeout CommandResult is returned
/// (exit_code set to a sentinel value, e.g., -1, stderr includes timeout message).
pub fn run_command(
    program: &str,
    args: &[&str],
    working_dir: Option<&Path>,
    env: &[(&str, &str)],
    timeout: Duration,
) -> std::io::Result<CommandResult>;
```

Implementation notes:
- Use `std::process::Command` with `stdout(Stdio::piped())`, `stderr(Stdio::piped())`.
- For timeout: spawn the child, then use a thread or `child.wait()` with polling in a loop checking elapsed time. If timeout exceeded, call `child.kill()`.
- Capture timing via `Instant::now()` / `elapsed()`.
- On Windows, `exit_code` from `ExitStatus::code()` may be None for signal-killed processes. Map None to -1.

### `gate/src/scratch.rs`

Scratch directory management.

```rust
/// Base path for all gate scratch dirs: `target/gate-scratch/`.
/// Derived from the workspace root (parent of gate crate dir, or from CARGO_MANIFEST_DIR).
pub fn scratch_base() -> PathBuf;

/// Create a timestamped run directory with per-stage subdirectories.
/// Returns the run directory path (e.g., `target/gate-scratch/run-20260417-143052/`).
/// Creates subdirectories: lot/, reel/, vault/, epic/
pub fn create_run_dir() -> std::io::Result<PathBuf>;

/// Delete the run directory. No-op if it doesn't exist.
pub fn cleanup_run_dir(path: &Path) -> std::io::Result<()>;
```

The run directory format is `run-YYYYMMDD-HHMMSS`. The per-stage subdirectories are created eagerly (all four, even if not all stages will run).

**Important (from CLAUDE.md):** Lot, reel, and epic stages need project-local scratch paths, NOT system temp. `target/gate-scratch/` satisfies this — it's under the workspace root, not under `%TEMP%` / `C:\Users`.

Register both modules in `main.rs`.

## TDD: Tests to Write First

### exec.rs tests

Use simple cross-platform commands for testing. On all platforms, `cargo` and `git` are available. For simpler tests, the gate binary itself can be used, or platform-appropriate commands.

**Cross-platform approach:** Create small Rust helper scripts is overkill. Instead:
- Use the actual `gate` binary (exit 0 from D1) for success case
- Use commands that exist everywhere: `git --version` (success), `git` with bad args (failure)
- For timeout testing: invoke a command that sleeps. Use `cargo` is too slow. Simplest: spawn a child that writes to stdout then blocks. Or on all platforms: use `ping` with a long count (Windows: `ping -n 10 127.0.0.1`, Unix: `sleep 5`). Use conditional compilation or runtime detection.

1. `capture_stdout` — run `git --version`, verify stdout contains "git version", exit_code is 0
2. `capture_stderr_on_failure` — run `git` with invalid subcommand, verify exit_code != 0, stderr non-empty
3. `exit_code_captured` — run `git --version`, verify exit_code == 0
4. `duration_recorded` — run any command, verify duration > 0
5. `working_dir_respected` — create a temp dir, run `git init` in it, verify it succeeds (the dir exists)
6. `env_vars_passed` — this is hard to test generically; verify env passing by running a command that uses an env var. Use conditional compilation: on Unix, `env` or `sh -c 'echo $FOO'`; on Windows, `cmd /c echo %FOO%`. Alternatively, skip this test and trust std::process::Command's env handling. **Recommendation:** write the test with platform-specific commands behind `cfg`.
7. `timeout_kills_process` — spawn a long-running command with a 1-second timeout. Verify it returns in ~1-2s, exit_code is -1 (or sentinel), stderr mentions timeout.
8. `timeout_not_triggered` — run a fast command with a 30s timeout. Verify it returns normally.

### scratch.rs tests

**Note (from CLAUDE.md):** Tests must use project-local paths, not system temp. Use `target/gate-scratch/` for test scratch dirs too. Clean up after each test.

9. `create_run_dir_exists` — create a run dir, verify it exists and matches the `run-YYYYMMDD-HHMMSS` pattern
10. `create_run_dir_has_subdirs` — verify lot/, reel/, vault/, epic/ subdirectories exist
11. `cleanup_removes_dir` — create a run dir, cleanup, verify it's gone
12. `cleanup_nonexistent_is_ok` — cleanup on a path that doesn't exist → no error
13. `consecutive_runs_unique` — create two run dirs in quick succession (may need a small sleep or fallback to appending a counter), verify different paths
