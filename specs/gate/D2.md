# D2: Assertion Helpers (`check.rs`)

## Context

Gate is a binary-only E2E test harness for the backlot workspace. Full design: `specs/GATE.md`. D1 established the crate scaffold, types, and CLI.

## Dependencies

D1 (types — `CommandResult`, `TestOutcome`).

## What to Build

### `gate/src/check.rs`

Assertion helper functions. Each prints a live `PASS: <label>` or `FAIL: <label>` line to stdout and returns `Result<(), TestFailure>`. Callers use `?` to early-return on failure.

```rust
pub struct TestFailure {
    pub label: String,
    pub detail: String,
}

impl std::fmt::Display for TestFailure {
    // "FAIL: {label}: {detail}"
}
```

Six assertion functions:

```rust
/// Asserts exit_code == 0.
pub fn assert_exit_ok(result: &CommandResult, label: &str) -> Result<(), TestFailure>;

/// Asserts exit_code != 0.
pub fn assert_exit_fail(result: &CommandResult, label: &str) -> Result<(), TestFailure>;

/// Parses stdout as JSON, asserts the top-level field exists and is non-null.
pub fn assert_json_field(json: &serde_json::Value, field: &str, label: &str) -> Result<(), TestFailure>;

/// Asserts haystack contains needle (case-sensitive).
pub fn assert_contains(haystack: &str, needle: &str, label: &str) -> Result<(), TestFailure>;

/// Asserts actual == expected using PartialEq. Debug-formats both on failure.
pub fn assert_eq<T: PartialEq + std::fmt::Debug>(actual: &T, expected: &T, label: &str) -> Result<(), TestFailure>;

/// Asserts the path exists on disk.
pub fn assert_path_exists(path: &Path, label: &str) -> Result<(), TestFailure>;
```

Each function:
1. Evaluates the condition
2. If pass: prints `PASS: {label}` to stdout, returns `Ok(())`
3. If fail: prints `FAIL: {label}: {detail}` to stdout, returns `Err(TestFailure { label, detail })`

The detail string should be specific. Examples:
- `assert_exit_ok`: "expected exit code 0, got 1"
- `assert_json_field`: "field 'usage' not found in JSON"
- `assert_contains`: "expected to find 'complete' in output (length 42)"
- `assert_eq`: "expected 5, got 3"
- `assert_path_exists`: "path '/tmp/foo.txt' does not exist"

Register the module in `main.rs`.

## TDD: Tests to Write First

All tests in `#[cfg(test)]` module within `check.rs`.

### Pass cases
1. `exit_ok_passes` — CommandResult with exit_code 0 → Ok
2. `exit_fail_passes` — CommandResult with exit_code 1 → Ok
3. `json_field_passes` — JSON object with the field present → Ok
4. `contains_passes` — haystack contains needle → Ok
5. `eq_passes` — two equal values → Ok
6. `path_exists_passes` — create a temp file, assert it exists → Ok

### Fail cases
7. `exit_ok_fails` — exit_code 1 → Err, detail mentions "expected exit code 0"
8. `exit_fail_fails` — exit_code 0 → Err, detail mentions "expected non-zero"
9. `json_field_missing` — field not in JSON → Err, detail mentions field name
10. `json_field_null` — field is JSON null → Err
11. `contains_fails` — needle not in haystack → Err, detail mentions needle
12. `eq_fails` — unequal values → Err, detail shows both values
13. `path_not_exists` — non-existent path → Err, detail shows path

### Edge cases
14. `contains_empty_needle` — empty needle always matches (passes)
15. `json_field_on_non_object` — JSON array/string → Err (not an object)
16. `exit_ok_negative_code` — exit_code -1 → Err (non-zero)

### Label propagation
17. `failure_contains_label` — verify `TestFailure.label` matches the label argument

## Acceptance Criteria

- `cargo test -p gate` — all D1 + D2 tests pass
- `cargo clippy -p gate -- -D warnings` clean
- Each assertion prints PASS or FAIL to stdout (visible in test output with `--nocapture`)
