# D1: Crate Scaffold, Types, CLI Parsing

## Context

Gate is a binary-only E2E test harness for the backlot workspace. Full design: `specs/GATE.md`. This is the first deliverable — no prior gate code exists.

## Dependencies

None (first deliverable).

## What to Build

### 1. Workspace registration

Add `"gate"` to `members` in the root `Cargo.toml`.

### 2. `gate/Cargo.toml`

Binary-only crate. No `lib.rs`, no library consumers.


### 3. `gate/src/main.rs`

Clap-based CLI. Parse args into `GateConfig`, print "gate: not yet implemented", exit 0. Wiring to the runner comes in D5.

### 4. `gate/src/types.rs`

All shared types used across modules:

```rust
/// Test stages in dependency order.
pub enum Stage { Flick, Lot, Reel, Vault, Epic, Mech }
```

Stage must implement: `Display`, `FromStr`, `Clone`, `Copy`, `PartialEq`, `Eq`, `PartialOrd`, `Ord`. `Display` is lowercase (`flick`, `lot`, ...). `FromStr` accepts lowercase, rejects anything else. Provide `Stage::all()` returning stages in dependency order and `Stage::default_timeout()` returning per-stage default timeout (`Duration`): 300s for all except Epic which is 600s.

```rust
pub enum TestOutcome {
    Pass,
    Fail(String),
    Skip(String),
    SoftFail(String),
}
```

Provide `is_pass()`, `is_failure()` (true only for `Fail`), `is_skip()`, `is_soft_fail()`.

```rust
pub struct TestResult {
    pub stage: Stage,
    pub test: String,
    pub outcome: TestOutcome,
    pub duration: Duration,
    pub cost_usd: Option<f64>,
    pub tokens_in: Option<u64>,
    pub tokens_out: Option<u64>,
}

pub struct StageResult {
    pub stage: Stage,
    pub results: Vec<TestResult>,
    pub duration: Duration,
}

pub struct CommandResult {
    pub stdout: String,
    pub stderr: String,
    pub exit_code: i32,
    pub duration: Duration,
}
```

`StageResult` should provide: `passed()`, `failed()`, `skipped()`, `soft_failed()` counts, `total_cost()`, `all_passed()` (true if zero `Fail` outcomes — `SoftFail` does not count as failure).

```rust
pub struct GateConfig {
    pub only: Option<Stage>,
    pub from: Option<Stage>,
    pub verbose: bool,
    pub bin_dir: Option<PathBuf>,
    pub timeout: Option<Duration>,
    pub output_dir: PathBuf,
    pub keep_scratch: bool,
}
```

`GateConfig` should provide `effective_timeout(stage: Stage) -> Duration` — returns explicit timeout if set, else the stage's default. Also `should_run(stage: Stage) -> bool` — respects `only` and `from` filters. Also `effective_keep_scratch() -> bool` — true if `keep_scratch` or `verbose`.

### 5. CLI definition

Clap derive struct in `main.rs` (or a `cli.rs` module). Flags per spec:

- `--only <stage>` — run exactly one stage
- `--from <stage>` — resume from a stage
- `--verbose` — save transcripts, implies keep-scratch
- `--bin-dir <path>` — override binary discovery
- `--timeout <seconds>` — per-stage wall-clock timeout
- `--output-dir <path>` — default: `gate/output/`
- `--keep-scratch` — preserve scratch directory

`--only` and `--from` are mutually exclusive (clap conflict).

## TDD: Tests to Write First

All tests go in `#[cfg(test)]` modules within the source files.

### types.rs tests

1. `stage_display` — each Stage variant displays as lowercase string
2. `stage_from_str_valid` — "flick", "lot", etc. parse correctly
3. `stage_from_str_invalid` — "unknown", "", "FLICK" all return Err
4. `stage_ordering` — `Stage::all()` returns `[Flick, Lot, Reel, Vault, Epic, Mech]`
5. `stage_default_timeout` — Epic is 600s, all others 300s
6. `test_outcome_classification` — `Pass.is_pass()` true, `Fail.is_failure()` true, `SoftFail.is_soft_fail()` true, `SoftFail.is_failure()` false
7. `stage_result_counts` — construct a StageResult with mixed outcomes, verify `passed()`, `failed()`, `skipped()`, `soft_failed()` counts
8. `stage_result_all_passed` — true when no Fail (SoftFail is OK), false when any Fail
9. `stage_result_total_cost` — sums `cost_usd` across results, skipping None
10. `gate_config_should_run_no_filter` — all stages run when no `only`/`from`
11. `gate_config_should_run_only` — only the specified stage runs
12. `gate_config_should_run_from` — stages before `from` are skipped, `from` and later run
13. `gate_config_effective_timeout_explicit` — explicit timeout overrides default
14. `gate_config_effective_timeout_default` — no explicit timeout returns stage default
15. `gate_config_effective_keep_scratch` — true when `verbose` even if `keep_scratch` is false

### main.rs / cli tests

16. `cli_defaults` — no args → default config values (output_dir = "gate/output/", verbose = false, etc.)
17. `cli_only_and_from_conflict` — both set → clap error
18. `cli_only_valid_stage` — `--only flick` parses correctly
19. `cli_invalid_stage` — `--only bogus` → clap error
20. `cli_timeout_parsed` — `--timeout 120` → Duration 120s
