Requirements:
- Columns right-aligned for numbers, left-aligned for stage name
- Cost formatted as `$X.XX` (two decimal places)
- Duration formatted as seconds with one decimal (`4.2s`), or minutes+seconds for >60s (`2m 5.3s`)
- Totals row sums all columns
- Empty results (no stages ran) в†’ still print header/footer with zero totals

#### JSON output

`pub fn write_results_json(results: &[StageResult], path: &Path) -> std::io::Result<()>`

Writes structured JSON. Format:

```json
{
  "stages": [
    {
      "stage": "flick",
      "tests": [
        {
          "test": "basic-invocation",
          "outcome": "pass",
          "duration_secs": 1.2,
          "cost_usd": 0.003,
          "tokens_in": 150,
          "tokens_out": 42
        }
      ],
      "duration_secs": 4.2,
      "total_cost_usd": 0.02
    }
  ],
  "total_tests": 27,
  "total_passed": 27,
  "total_failed": 0,
  "total_skipped": 0,
  "total_soft_failed": 0,
  "total_cost_usd": 1.52,
  "total_duration_secs": 139.1
}
```

Outcome serialized as: `"pass"`, `"fail"`, `"skip"`, `"soft_fail"` (the detail string is a separate `detail` field, present only for non-pass).

#### Cost/duration helpers

`pub fn total_cost(results: &[StageResult]) -> f64` вЂ” sum of all test costs, treating None as 0.

`pub fn total_duration(results: &[StageResult]) -> Duration` вЂ” sum of stage durations.

Register the module in `main.rs`.
